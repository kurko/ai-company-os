# Software Craft

Neural triggers for code quality, testing, and object-oriented design. Reference these to activate trained knowledge.

## Books

### Clean Code
**Author:** Robert C. Martin
**Trigger phrase:** "Apply Clean Code principles for..."
**Key concepts:**
- Meaningful names that reveal intent
- Functions should do one thing
- Comments are a failure to express in code
- The Boy Scout Rule: leave code cleaner than you found it
- Error handling is one thing

**When to invoke:** Naming decisions, method extraction, code readability reviews, refactoring for clarity.

---

### Growing Object-Oriented Software, Guided by Tests (GOOS)
**Author:** Steve Freeman & Nat Pryce
**Trigger phrase:** "Apply GOOS principles for..."
**Key concepts:**
- Test-first, outside-in development
- Walking skeleton: get end-to-end working first
- Listen to your tests - pain indicates design problems
- Ports and adapters for external dependencies
- Mock roles, not objects

**When to invoke:** Starting new features, designing test strategy, dealing with external dependencies, TDD approach decisions.

---

### Practical Object-Oriented Design in Ruby (POODR)
**Author:** Sandi Metz
**Trigger phrase:** "Apply Sandi Metz's design principles for..."
**Key concepts:**
- Single Responsibility: a class should have one reason to change
- Depend on abstractions, not concretions
- Prefer composition over inheritance
- Duck typing and polymorphism
- The cost of dependencies

**When to invoke:** Class design, dependency management, inheritance vs composition decisions, interface design.

---

### 99 Bottles of OOP
**Author:** Sandi Metz & Katrina Owen
**Trigger phrase:** "Apply 99 Bottles refactoring approach for..."
**Key concepts:**
- Shameless Green: make it work first, simply
- Open/Closed: open for extension, closed for modification
- Flocking Rules: find the smallest difference
- Refactor under green tests
- Reach for abstractions only when needed

**When to invoke:** Refactoring decisions, when to abstract, incremental improvement, dealing with duplication.

---

### Refactoring
**Author:** Martin Fowler
**Trigger phrase:** "Apply Refactoring catalog for..."
**Key concepts:**
- Refactoring: changing structure without changing behavior
- Code smells as indicators
- Small, reversible steps
- Refactor before adding features
- Extract Method, Extract Class, Move Method

**When to invoke:** Identifying code smells, choosing refactoring techniques, preparing code for new features.
